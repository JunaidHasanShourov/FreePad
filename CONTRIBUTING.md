# Contributing to FreePad

Thank you for your interest in contributing to FreePad! We welcome all kinds of contributions, from bug reports to new features.

## Code of Conduct

Please be respectful and constructive in all interactions. We're building a community for everyone.

## How to Contribute

### 1. Report Bugs

If you find a bug, please open an issue with:
- Clear description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser/OS information
- Screenshots if relevant

### 2. Suggest Enhancements

Have an idea for improvement? Open an issue with:
- Clear description of the enhancement
- Use cases and benefits
- Possible implementation approach
- Any relevant examples

### 3. Submit Code Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your branch
7. Open a Pull Request

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/freepad.git
cd freepad

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

## Code Style Guidelines

- Use functional components with React hooks
- Keep functions small and focused
- Use meaningful variable names
- Add comments for complex logic
- Follow existing file structure
- Use CSS variables for colors and spacing

## Commit Message Format

Use clear, descriptive commit messages:

```
git commit -m "Add/Fix/Update: Brief description"

# Examples:
git commit -m "Add: Dark mode toggle"
git commit -m "Fix: Export to PDF button not working"
git commit -m "Update: Improve drawing performance"
```

## Pull Request Process

1. Ensure your PR title clearly describes the change
2. Add a description of what was changed and why
3. Reference any related issues
4. Test on multiple browsers if possible
5. Wait for review and feedback
6. Make requested changes
7. Once approved, the maintainers will merge

## Testing Checklist

Before submitting a PR, test:
- [ ] All existing features still work
- [ ] New feature works as intended
- [ ] No console errors
- [ ] Responsive on desktop and tablet
- [ ] Export functionality works
- [ ] Auto-save is functioning

## Build & Deployment

Make sure your changes:
- Build successfully: `npm run build`
- Pass linting: `npm run lint`
- Have no console warnings
- Work in production build

## Questions?

Feel free to:
- Open a discussion on GitHub
- Ask in the issues
- Review existing documentation
